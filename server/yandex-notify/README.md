# Уведомления о записках

Этот шаблон нужен для запуска отправки заявок на почту, в Telegram и VK без раскрытия секретов в браузере.

Сайт отправляет JSON на `VITE_ORDER_ENDPOINT`. Функция принимает заявку, валидирует ее и рассылает уведомления.

## Почему нужен backend

Нельзя отправлять заявки напрямую из React-сайта в Telegram, VK или SMTP: токены бота, ключ VK и пароль почты будут видны в исходном коде сайта. Поэтому секреты должны храниться только в серверной функции.

## Дешевый вариант запуска

1. Развернуть эту папку как Node.js функцию в Yandex Cloud Functions или другом российском serverless/хостинге.
2. Установить переменные окружения из `.env.example`.
3. В GitHub repo settings добавить repository variable `VITE_ORDER_ENDPOINT` со ссылкой на публичный HTTP endpoint функции.
4. Проверить тестовую записку.

## Локальная проверка без секретов

```powershell
npm install
npm run test:dry
```

Для локального HTTP endpoint:

```powershell
$env:DRY_RUN="true"
$env:ALLOWED_ORIGIN="http://127.0.0.1:5173"
npm start
```

После этого можно собрать сайт с:

```text
VITE_ORDER_ENDPOINT=http://127.0.0.1:8787
```

## Каналы

- Email получает полный текст записки.
- Telegram и VK по умолчанию получают краткое уведомление без полного списка имен.
- Если нужно отправлять полные имена и в мессенджеры, поставить `SEND_FULL_TO_MESSENGERS=true`, но это лучше согласовать отдельно из-за персональных данных.

## VK

Для VK нужен токен сообщества с правом отправки сообщений. Сообщество должно разрешать сообщения, а получатель или беседа должны позволять сообщения от сообщества.

## Telegram

Создается бот через BotFather. Бота добавляют в закрытую группу ответственных, затем получают `TELEGRAM_CHAT_ID` этой группы.

Публичная ссылка `https://t.me/BlagoVhram` нужна для контактов на сайте, но сама по себе не позволяет отправлять заявки. Для автоматических уведомлений нужен бот и `TELEGRAM_CHAT_ID`.

## Почта

Для SMTP лучше использовать почту на домене храма или отдельный служебный ящик. Для Яндекс 360 обычно нужен пароль приложения, а не основной пароль аккаунта.

## Защита от мусора

В frontend есть honeypot-поле `website`, функция валидирует сумму, номер заявки и список имен, а также проверяет `Origin` по `ALLOWED_ORIGIN`. Для боевого запуска позже можно добавить CAPTCHA или ограничение частоты запросов.
